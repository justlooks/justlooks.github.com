Jul 3rd, 2012

1. 首先配置getmail收信

安装getmail4

apt-get install getmail4

在非root用户目录下创建邮箱

mkdir -m 700 ~/Mail/inbox/{,tmp,new,cur}

创建配置文件

mkdir -p ~/.getmail;cd ~/.getmail
vi rc-gmail

添加内容

测试能够正常收信后配置完成

如果需要配置多个帐号的话,那么需要写多个rc文件,一个rc文件对应一个邮箱帐号,运行时,通过-r选项传入多个配置文件

getmail -r rc-gmail -r rc-company

如果需要每隔10分钟收信一次则可以列入计划任务

那么现在所有的邮件都是放到一个目录下的,大杂烩,如何对这些邮件进行分类筛选呢?
比如现在的需求是需要将@company.com来源的邮件放入Mail中的workmate目录下,
而将公司行政通知(notify@company.com)放到Mail中的notify目录下

2. 配置procmail对邮件进行过滤

安装procmail

apt-get install procmail

创建配置文件

vi ~/.procmailrc

添加内容

创建对应目录

mkdir -p ~/Mail/{workmates,notify}

同时需要对getmail对应的配置文件做些修改

[destination]
type = Maildir
path = ~/Mail/inbox/

改为

[destination]
type = MDA_external
path = /usr/bin/procmail
unixfrom = true

再运行getmail时就会发现邮件被正确的放到对应的目录下了
